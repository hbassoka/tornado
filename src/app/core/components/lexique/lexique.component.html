Lexique

Quelques termes essentiels pour mieux comprendre le jargon IT

1. API

Application Programming Interface : ensemble de règles permettant à des applications de communiquer.

2. Algorithme

Suite d’instructions permettant de résoudre un problème ou d’effectuer une tâche.

3. Backend

Partie serveur d’une application qui gère les données et la logique métier.

4. Base de données

Système organisé pour stocker, gérer et récupérer des données.

5. Bug

Défaut ou erreur dans un logiciel provoquant un comportement inattendu.

6. Cloud

Serveurs distants sur Internet pour stocker et gérer des données.

7. CI/CD

Continuous Integration / Continuous Deployment : automatisation des tests, intégration et déploiement du code.

8. CSS

Cascading Style Sheets : langage de style pour formater l’apparence des pages web.

9. DevOps

Pratiques combinant développement logiciel et opérations IT pour améliorer la livraison continue.

10. DNS

Domain Name System : traduit les noms de domaine en adresses IP.

11. Frontend

Partie visible d’une application avec laquelle l’utilisateur interagit.

12. HTML

HyperText Markup Language : langage de balisage pour structurer une page web.

13. HTTP

HyperText Transfer Protocol : protocole pour transférer des données sur le web.

14. HTTPS

Version sécurisée de HTTP avec chiffrement SSL/TLS.

15. JavaScript

Langage de programmation côté client pour rendre les pages web interactives.

16. JSON

JavaScript Object Notation : format léger pour échanger des données entre applications.

17. Middleware

Logiciel intermédiaire gérant la communication entre applications et serveurs.

18. Microservices

Architecture découpant une application en services indépendants et spécialisés.

19. Node.js

Environnement JavaScript côté serveur pour créer des applications réseau rapides.

20. ORM

Object-Relational Mapping : technique qui relie objets et base de données relationnelle.

21. Proxy

Serveur intermédiaire entre client et serveur pour filtrer ou cacher les requêtes.

22. REST

Representational State Transfer : style d’architecture pour créer des API.

23. SaaS

Software as a Service : logiciel accessible via Internet sans installation locale.

24. SDK

Software Development Kit : ensemble d’outils pour développer des applications pour une plateforme.

25. SEO

Search Engine Optimization : optimisation d’un site pour le référencement naturel.

26. SQL

Structured Query Language : langage pour interroger et gérer des bases de données relationnelles.

27. SSL

Secure Sockets Layer : protocole pour sécuriser les communications Internet.

28. Token

Jeton numérique utilisé pour authentifier ou valider des transactions.

29. UI

User Interface : interface utilisateur d’une application ou d’un site web.

30. UX

User Experience : expérience globale de l’utilisateur lors de l’utilisation d’une application.

31. Virtualisation

Création de ressources virtuelles à partir de ressources physiques.

32. VPN

Virtual Private Network : réseau privé sécurisé sur Internet.

33. WebSocket

Protocole pour communication bidirectionnelle en temps réel entre client et serveur.

34. Versioning

Gestion des différentes versions d’un logiciel ou d’un document.

35. Virtual DOM

Représentation en mémoire de l’interface pour optimiser le rendu dans des frameworks comme React.

36. Container

Environnement isolé permettant d’exécuter une application avec toutes ses dépendances.

37. Framework

Ensemble de bibliothèques et outils facilitant le développement d’applications.

38. Git

Système de gestion de versions distribué pour suivre l’historique du code.

39. GitHub

Plateforme hébergeant des projets Git et facilitant la collaboration.

40. HTTPS

Protocole sécurisé pour transférer des données sur Internet (répétition volontaire pour insister sur sécurité).

41. IDE

Integrated Development Environment : logiciel pour écrire, tester et déboguer du code.

42. JSON Web Token

Format sécurisé pour transmettre des informations entre un client et un serveur.

43. Load Balancer

Répartiteur de charge qui distribue le trafic entre plusieurs serveurs.

44. Middleware

Logiciel intermédiaire gérant la communication entre applications (répété pour insister sur usage).

45. ORM

Technique liant objets et base de données relationnelle (répété pour rappel).

46. PaaS

Platform as a Service : plateforme cloud pour développer et déployer des applications.

47. React

Bibliothèque JavaScript pour créer des interfaces utilisateur dynamiques.

48. RESTful

Style d’architecture d’API respectant les principes REST.

49. Router

Composant ou service gérant la navigation entre pages d’une application web.

50. SaaS

Logiciel accessible en ligne sans installation locale (répété pour rappeler concept).

51. Scrum

Méthode agile pour gérer des projets logiciels avec des itérations courtes et des réunions quotidiennes.

52. SQL Injection

Technique d’attaque exploitant les vulnérabilités d’une base de données via des requêtes SQL malveillantes.

53. SaaS

Logiciel accessible via Internet, hébergé et maintenu par un fournisseur.

54. SDK

Ensemble d’outils et bibliothèques pour développer sur une plateforme spécifique.

55. Serverless

Architecture où le fournisseur cloud gère automatiquement le serveur et l’exécution des fonctions.

56. SOAP

Protocole pour échanger des informations structurées via XML entre services web.

57. Spring Boot

Framework Java facilitant la création d’applications backend autonomes et prêtes à l’emploi.

58. SSL/TLS

Protocoles pour sécuriser les communications sur Internet par chiffrement.

59. Test Unitaire

Méthode de test consistant à vérifier le fonctionnement d’une unité de code isolée.

60. Test d’Intégration

Test vérifiant l’interaction correcte entre plusieurs modules ou composants.

61. UI/UX Design

Conception de l’interface utilisateur et de l’expérience globale pour les utilisateurs.

62. URL

Uniform Resource Locator : adresse web permettant de localiser une ressource sur Internet.

63. UX

User Experience : expérience et satisfaction globale de l’utilisateur lors de l’utilisation d’une application.

64. Version Control

Système de gestion des versions du code pour suivre les modifications et collaborer efficacement.

65. Virtual Machine

Environnement logiciel simulant un ordinateur physique pour exécuter des systèmes ou applications.

66. Web API

Interface de programmation accessible via HTTP pour échanger des données avec une application web.

67. Webhook

Mécanisme permettant à une application de recevoir automatiquement des notifications d’un service externe.

68. WebSocket

Protocole pour communication bidirectionnelle en temps réel entre client et serveur.

69. XML

eXtensible Markup Language : format de données structuré utilisé pour échanger des informations.

70. YAML

Yet Another Markup Language : format de données lisible utilisé pour la configuration et les échanges.

71. Agile

Méthodologie de développement itérative et collaborative favorisant l’adaptation au changement.

72. Big Data

Traitement et analyse de très grands volumes de données complexes.

73. Blockchain

Base de données distribuée et sécurisée utilisée pour enregistrer des transactions.

74. Bot

Programme automatisé qui effectue des tâches répétitives sur Internet.

75. CDN

Content Delivery Network : réseau de serveurs pour distribuer du contenu plus rapidement.

76. Cybersecurity

Ensemble de pratiques pour protéger les systèmes et données contre les cyberattaques.

77. Data Lake

Réservoir centralisé pour stocker des données brutes de toutes sortes.

78. Data Warehouse

Système centralisé pour stocker et organiser les données pour l’analyse.

79. Docker

Plateforme pour créer, déployer et exécuter des applications dans des containers.

80. Encryption

Processus de chiffrement des données pour les sécuriser contre les accès non autorisés.

81. Firewall

Barrière de sécurité qui contrôle le trafic réseau entrant et sortant.

82. GitLab

Plateforme Git pour héberger du code et gérer des projets DevOps.

83. GraphQL

Langage de requête pour API permettant de demander exactement les données nécessaires.

84. Hash

Fonction qui transforme des données en une chaîne fixe pour vérification ou sécurité.

85. IoT

Internet of Things : objets connectés communiquant entre eux via Internet.

86. JWT

JSON Web Token : standard pour transmettre des informations sécurisées entre parties.

87. Kubernetes

Plateforme open-source pour orchestrer des containers à grande échelle.

88. Load Balancer

Répartisseur de charge distribuant le trafic entre plusieurs serveurs.

89. Machine Learning

Technique d’IA permettant à une machine d’apprendre à partir de données.

90. Middleware

Logiciel intermédiaire entre applications et services (répété pour rappel).

91. Multithreading

Exécution simultanée de plusieurs threads pour améliorer les performances.

92. NoSQL

Bases de données non relationnelles adaptées aux grandes volumétries et aux données non structurées.

93. OAuth

Protocole d’autorisation sécurisé permettant à un utilisateur de donner accès à ses données sans partager son mot de passe.

94. Open Source

Logiciel dont le code source est librement accessible et modifiable.

95. PHP

Langage de programmation serveur pour créer des pages web dynamiques.

96. Python

Langage polyvalent utilisé pour le web, la data science et l’IA.

97. React Native

Framework pour développer des applications mobiles multiplateformes avec React.

98. SaaS

Logiciel disponible en ligne sans installation locale (répété pour cohérence).

99. TypeScript

Langage basé sur JavaScript avec typage statique, utilisé notamment avec Angular.

100. WebAssembly

Format binaire permettant d’exécuter du code proche du natif dans le navigateur.
